Disabling the AHOI Module

Note: The AHOI Module is already disabled in ZoomText version 10, so the following solution is for versions previous to 10.

ZoomText fails to track and speak items in the Case Management screens; there are also issues with other ZoomText features such as Mouse Echo.

Current Solution:

To disable ZoomText's AHOI processing for a specific Web site:

  1. On the Windows Desktop, right-click on the ZoomText program icon or use the hot key SHIFT+F10.
  2. In the pop-up application menu select Properties. The ZoomText 9.1 Properties dialog opens.
  3. Select the Find Target button. (Windows 7 users select the Open File Location button. Windows Explorer opens ZoomText’s program folder.
  4. Locate and open the file: AHOI\ah_ie_punt_url.txt. This opens the file in a text editor such as Windows Notepad.
  5. Add the domain (URL) for the System 7™ web site as a new line in the file. For example, "https://application.libera.com/..." The inclusion of the forward slash and three periods at the end of the URL tells ZoomText to disable the AHOI module for all pages nested within the directories of the defined URL as well as specialized processing throughout the Web site.
  6. In the File menu, select the Save option.
  7. Exit the text editor application.
  8. Exit and restart ZoomText and Internet Explorer.

Note: For more information, visit the Web site of AISquared, makers of ZoomText: http://www.aisquared.com/Support/.
