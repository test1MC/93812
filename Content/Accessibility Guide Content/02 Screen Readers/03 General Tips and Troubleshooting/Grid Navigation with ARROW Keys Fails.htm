Grid Navigation with ARROW Keys Fails

Within a System 7™ grid, if you are unable to use the ARROW keys to navigate, the screen reader may need to be configured. Alternatively, this may be an issue with an Internet Explorer setting; this is the case if the phrase “System 7” is not included in Internet Explorer window titles.

Current Solution:

Please walk through the steps outlined for your specific screen reader in either the Preparing JAWS® or the Preparing Window-Eyes® section to double-check that your screen reader is set up correctly.

Alternate Solution:

Enable script initiated windows in Internet Explorer for Trusted Sites.

For Internet Explorer 7.0 or greater:

  1. Open Internet Explorer.
  2. Open Internet Options. This option is available via the Tools menu if available or via the Tools button ( ).
  3. Select Internet Options…
  4. Go to the Security tab.
  5. Select Trusted Sites from the Select a Web Content Zone to Specify Its Security box.
  6. Select the Sites… button. This opens the Trusted Sites box.

    Internet Explorer Options. 1) Security Tab. 2) Trusted Sites. 3) Custom level button.
  7. In the Custom Level dialog, under Miscellaneous select the Allow script initiated windows without size or position constraints setting and set it to Enable.

    Security Settings dialog with Allow script-initiated windows without size or position constraints selected.

    Without this setting enabled, Internet Explorer adds part of the URL to the window title. This confuses the screen readers and disables the System 7 screen reader scripts. The screen readers require that the text "System 7" is visible in the window title. With the URL added by Internet Explorer, this sometimes pushes the "System 7" phrase out of the visible window title.
