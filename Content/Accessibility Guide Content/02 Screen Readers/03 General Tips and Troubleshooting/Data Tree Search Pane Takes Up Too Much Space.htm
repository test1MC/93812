Data Tree Search Pane Takes Up Too Much Space

The Data Tree Search Pane is taking up a great deal of space thereby restricting the grid view. The Search Pane is the yellow section at the top of the Data Tree that allows you to search the contents of the grid. It's really only helpful when you're working with grids that have parent and child tables because you can search the child tables via the Search Pane.

Current Solution:

Set your System 7™ User Preference for Show Data Tree Search Pane to No. After you log out and back into the system, the Search Pane will then no longer be shown and more available grid records will be visible.


Example of the Data Tree with and without the Search Pane.

In the example from the image above removing the Search Pane changed the number of visible records (without scrolling) from five to eight.

The size of the font within the grid also affects the amount of visible records in grids. The larger the grid zoom percentage, the less rows appear in the visible window. This means you'll have to scroll more. Changing the grid zoom also affects column widths. The User Preference setting that changes the font size in grids is Default grid zoom percentage?


Example of the difference between 130% and 100% grid zoom percentages.

In the example from the image above changing the grid zoom percentage from 130% to 100% changed the number of visible records (without scrolling) from eight to 11.
