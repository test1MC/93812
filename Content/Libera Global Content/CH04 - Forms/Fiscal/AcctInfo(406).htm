Account Information


Top section of the Account Information form.

Overview

The form shows information about budget accounts used to charge and pay for goods and services. Agencies track funds allocated to various programs via budget accounts. This form is available when the PDQ is run and you selects the Create Account button or by selecting an existing account record in the Data Tree.

Form Fields

Many of the form fields and related data located in the top section of the form are copied into the various s associated with the account. The ability to edit a field varies on the section in which it appears.

  • Office* – Shows the name of the office where the account is assigned. This field is editable for level three s or if your Agency is not using s. The numeric code for the office becomes part of the account number.
  • Fiscal Year* – This value becomes part of the account number.
  • Account Name – The account name entered here appears in lookups/picklists and forms.
  • Account Identifier Group – Shows the title of the Account Identity Group associated with the Account Identifier.
  • Program – This associates the account with an agency program.
  • County -This associates the account with a county.

 

Account Ranking


Account Ranking section.

This section appears when the Account Type is Authorization or Upper Level Account. It shows information about special target groups that apply to the account. This section is only editable for first level accounts. You can select multiple Special Project choices at one time.

When authorizing services for s authorizations will select accounts where the Special Project (on the form) matches the ’s Special Project (on the Application and Intake form) or the Default Project Description for the ’s program. Default Project Descriptions are set for programs in the Dictionary Table Administration module, specifically the Account Special Project Codes table.

When multiple accounts match, then the account is selected in Rank order (from 1 to infinity). If a value is not entered for Rank, the account's rank is assumed to be 100. In that event, it will be the last account used for authorizations.

Special Project and Rank are not considered if the vendor’s service is contracted and an account is specified for that contract or when processing Agency authorizations.

Use the Only available if the service specifies this Account Identifier or Account Number? field to further control when this account can be used. When this field is selected this account can only be used when the Account Identifier or Account Number matches the requirements for the service being provided.

Sub Accounts section of the Account Information form.

When you set up a budget account you have the ability to create a hierarchy of accounts. Think of s as a way to subdivide money between organizational units. Total funds are added to the Level 1 account, then allocated as desired to all related s. Although account activity is tracked at the authorization account level, the s show the totals of all of their s.

The hierarchy can have up to three levels. Each level may have multiple accounts. For example:

Level 1 = , ie. Agency wide funds.

Level 2 = Child Account, ie. funds, create one for each .

Level 3 = Grandchild Account, ie. Office funds, create one for each Office in the .

s are created one of two ways. They can be created via the section of the Level 1 or 2 account or, you can select Level 1 or 2 account in the Data Tree, then select the Create button.

Other Miscellaneous Information about s

  • Agencies are not required to use s.
  • s are displayed in the Data Tree grouped under the Level 1 account.

 Steps to Create Accounts with s From the Ground Up!

 

Adding the Upper Level Account

  1. Run the PDQ.
  2. In the Data Tree, select the Create an Account button. Use your mouse, or with cursor focus in the Data Tree, press CTRL+SHIFT+B to jump to the buttons at the bottom of the Data Tree. This opens the form.
  3. In the form for the Account Type field, select Upper Level Account from the lookup.
  4. Enter the Fiscal Year (FY). This becomes part of the Account Number.
  5. Add the Segment of Year. This becomes part of the Account Number.
  6. Select the Program. This causes the Account Identifier to populate which in turn becomes part of the Account Number.
  7. Enter a Begin and End Date for the account. Usually these correspond with the FY you are using. For example if your FY is 2016 and you are using the Federal FY the Begin Date would be 10/01/2015 and the End Date would be 09/30/2016.
  8. Set the Turn Account Off? field to N (No). Otherwise no one will be able to use the account and all related s.
  9. Select the Special Project for the account.
  10. Enter a number (1-99) for the Rank. The higher the number means that the account will be used more frequently. If left blank, the account's rank is assumed to be 100. It will be the absolutely last account ever used.
  11. Enter the amount. Now you're ready to add a Level 2 .

Adding a Level 3 Account

  1. Close the Level 1 form that has the Level 2 that you just added.
  2. When the Data Tree refreshes you will see your newly created Level 1 account listed with a plus sign to the left of the Account Name. Select the account and expand the row. Single-click the plus sign, or with cursor focus on the row press the SPACE bar to expand the row.
  3. When the row expands you will see the Level 2 account(s) listed under the Level 1 account. Select the Level 2 account you just created to open its own form.
