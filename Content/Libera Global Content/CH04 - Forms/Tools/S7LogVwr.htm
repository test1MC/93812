System 7 Log Viewer

Accessing Tools

 


System 7 Log Viewer.

 

The System 7 Log Viewer is used by System Administrators to retrieve log file data for system activity. It opens in the System 7 Administration Console. This is especially helpful when troubleshooting.

 

Log Viewer Fields Defined

  • Server – Defaults to a Web Server option. Some agencies have multiple web and application servers. Open the drop down and select an application server. If there are multiple application servers you may have to work through each one to find anything useful. The App Server holds the app log and the SQL log. The Web Server holds the UI log.
  • Log File – Defaults to Application Log which is what we commonly use, however you can also use the SQL Trace log – just know that it’s exponentially larger and takes longer to load. The Application Log shows what’s happening on the application side. The SQL Trace log shows what’s happening on the database side, for example it will show the SQL for a PDQ.
  • Start Date/Time – Defaults to Today’s date. To reduce the number of log files returned (and speed up your search exponentially) enter the time immediately following the date. If you get an error window it may include a time stamp, go a minute before that time. The time stamp does include milliseconds, if you don’t have that information just fill in zeroes. The time stamp understands military and non-military time.

    Time entered following the date.
  • End Date/Time – Use if you’re looking within a specific time range before ‘now’. If you’re looking for something that just happened you can leave this field blank.
  • View Log – This button loads the log file data in the window below the fields.
  • Download Log – This button downloads the log file data to a text file. Choose to "Save As" so you have the file name and location. This file can be difficult to read, because the main users are developers. The log file can be extremely large. Tip - You can copy and paste text from the log into another text editor or word processor for viewing.
  • Clear Log – This button literally clears the log. It's strongly recommended that you do not do this. If in doubt talk to your Supervisor.
  • Start Line – Not really used, although you could guess at the start line number and enter it if you wanted. Seriously though, if you’re using the Start Date/Time this field isn’t needed.
  • Number of Lines – The number of lines returned has a max limit of something like 500 or 1000 lines. It’s conceivable that even for a short period of time, for example, a half hour, that the number of lines could exceed the max limit. Enter ‘-1’ to insure that *all* lines within your time span (either the Start Date/Time to End Date/Time or Start Date/Time to Now) are returned.
  • Email Log – Opens the Send To window that allows you to send the log file data outside of the system. Always keep in mind client confidentiality as the log file data (both application and SQL Trace) may include SSNs and other identifying information. This method is not secure. It’s OK for internal use from internal environments. If in doubt, check with your Supervisor. The email shows the log file data the same as it’s displayed in the log viewer window.
  • Log Marker – Use if you want to create a marker, think something like a bookmark within the log file data.
  • Create Log Marker – This button inserts the Log Marker you entered into the log file data, for example:

<logevent datetime="2014/06/03 16:53:30.632" action="Log.CreateMarker" severity="Informational" eventid="e1e2b85c-9b9a-4a75-9b7f-4fcad927c14b" user="983" marker="MONICA" requesturl="http://localhost/Sys7WS-MO/AdminWS.asmx" remoteip="::1" requestid="e1e2b85c-9b9a-4a75-9b7f-4fcad927c14b" >

<message><![CDATA[

Log Marker: MONICA

]]>

</message>

</logevent>

 

 

All tips below are optional. These apply to both the Application Log and the SQL Trace Log.

  • It’s best if you can recreate the error then run the log viewer immediately. It keeps the number of lines returned to a low number.
  • Use the Find feature, CTRL+F, to search for the word ‘error’ or “error” (with the double quote), or the exception, client number/name, your name, form template ID, component ID #, etc.
  • It’s not uncommon for errors to be the longest logevents. Usually if you scroll through the lines returned you can find it.
  • Copy the lines returned into word where you can format the text to draw attention to the data you want the Libera developers to see.
  • When copying make sure you copy from <logevent datetime=”YYYY/MM/DD HH:MM:SS…> to </logevent> and to include all logevents with the same date/time stamp.
