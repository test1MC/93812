Vendor Validation Dialog.

 

Overview

The Vendor Validation Dialog is used to search for vendors and to add new vendors to the system. The Vendor Validation Dialog is the first step in adding a new Vendor to the system. The Vendor Validation Dialog is opened by activating the Search / Add Vendor button at the bottom of the Data Tree when viewing vendors. This button is available via vendor PDQs as configured by your Agency. Use your mouse to select the button or press CTRL+SHIFT+B to jump down to the buttons, then press the DOWN ARROW key to move to the Search / Add Vendor button. When cursor focus is on the button, press ENTER.

 

 

Form Fields

When adding a new vendor, all fields entered below will appear on the Vendor Information form.

 

  • Name - Type the name of the vendor you want to find or add to the vendor list. The name you enter is searched against existing data in the system (Vendor Legal Name, Search By, Doing Business As Name, First Name, Last Name all on the form. The search results displayed in the grid may display vendor names that do not match what you entered in the Name field of the Vendor Validation Dialog as a result. To find out why a vendor appears in the list, scroll or RIGHT ARROW to view all of the columns available.
  • T.I.N - Tax payer Identification Number. This is the identification number used by the Internal Revenue Service (IRS). A TIN may be the Social Security Number of the sole proprietor or the Employer Identification Number (EIN). To add a new vendor you must include a TIN. Both numbers are nine digits, but they are formatted slightly differently: Example EIN: ##-####### or Example SSN: ###-##-####. If you enter a number that does not meet the formatting requirements, the system displays a message letting you know. If you get this message, reenter the number and try again.
  • Phone - Phone number fields are formatted to include the area code. Always include the area code when entering a phone number.
  • Street - First line of the vendors street address. Type the building number and street name.
  • Zip - You can type the Zip Code or use the lookup to select from a list. Regardless of how you add the Zip Code, adding it also adds the city name associated with it. In some instances, you will find that one zip code may have many city names associated with it. When this happens the lookup will open, filtered to show you all of the names associated with the zip code you entered. Select one of the city names from the lookup by double-clicking or using the DOWN ARROW key to move to it and then press ENTER.
  • City - This field automatically populated with selection of Zip. However, you may edit it if necessary.
  • Search for All Possible Matches and/or Search for Exact Matches – Use these buttons to start the search process. There are two search buttons available. By default all users see the Search for All Possible Matches button. System Administrators also see the Search for Exact Matches button. Your Agency may choose to have Search for Exact Matches displayed for all users. System Administrators have the ability to display the second button. Possible Matches are Vendors that match at least one of the fields. Exact Matches must match all fields that aren't left blank.

 

 How to Display the Search for Exact Matches Button
  1. Log on with System Administrator rights.
  2. Select System Administration from the Application Home Page.
  3. In Group: General select Application Settings. Press ENTER to load the Electronic Case Folder (ECF) with the Application Settings grid. Then press CTRL+SHIFT+T to move cursor focus to the Filter Row.
  4. In the Name cell of the Filter Row type the word search and then press ENTER.
  5. The filter results will show one option: Show the Search for Exact Match button on Client Validation based on Roles. Note: This works with the Vendor Validation Dialog as well.
  6. Select the option to open the Application Setting Form.
  7. Show the Search for Exact Match button on Client Validation based on Roles – this is the only field on the form. Type Role ID numbers in a comma separated list that are allowed to see the Exact Match button. You have three options for editing.
    1. All users can see the Exact Match button – Clear the existing value (24) from the cell.
    2. Only select users can see the Exact Match button – Enter the Role ID numbers (found in Role Administration under Group: Security) of the roles that are allowed to see the button in the cell. Type the numbers separated by commas, no spaces and do not end with a comma. It is not necessary to list them in numeric order. For example: 24,2,3,1000,6.
    3. No users can see the Exact Match button – Enter !0 in the cell.
  8. After editing the user roles allowed to see the Exact Match button, close the form.

To see your change, log on with one of the user roles you added (or have another user with one of the roles specified). Load a PDQ that allows you to add a new client or a new vendor. Then select the Search / Add Client/Vendor button at the bottom of the Data Tree. The Client or Vendor Validation Dialog opens displaying both search buttons.

 

 Results Grid

Once you have filled out at least the minimum amount of fields and select a search button, if any existing vendor in the system matches any of the data entered, that vendor (or vendors) is shown in the grid portion of the Vendor Validation Dialog.

The grid here works exactly the same as grids throughout the system with one notable exception. Normally you can use the TAB and SHIFT+TAB keys to move to the right and left in grids. In the results grid this will not work, tabbing will move you to the next field on the form. Instead use the RIGHT and LEFT ARROW keys to move right and left in the grid.

  • Utilizing the check box next to the selected vendor opens the Vendor Information form.
  • Double clicking on the selected vendor without a check mark, opens to the View Vendor PDQ in the Data Tree.
  • Refer to Grid Functionality for information on working with grids.

 

 Adding a New Vendor
  1. Add a new Location for an existing Vendor (Optional)? - When you use this option, you are not required to fill in any of the fields on the Vendor Validation Dialog. Use this field to link a new location to an existing vendor. Good examples for this include national and local chains.
    1. Click the blue pull down arrow or press ALT+F1 to open the lookup.
    2. The lookup window shows all of the existing vendors in the system. Use the Filter Row to quickly search the list for the vendor you want. Then double-click the vendor name, or press the DOWN ARROW key until you reach the name and then press the ENTER key. This action closes the lookup window and displays your selection in the field.
    3. Click or TAB to and press ENTER on the Add as a new Location button. This closes the Vendor Validation Dialog and opens the Vendor Information form. The form is preloaded with some of the data from the existing vendor such as Vendor Legal Name and TIN. You will need to fill in information specific to the new location.
  2. Add as New Vendor - Single-click or press the SPACEBAR to activate this button. This action closes the Vendor Validation Dialog and opens a new Vendor Information form. The form will be blank with the exception of the fields you entered on the Vendor Validation Dialog. Note: You will not be able to add a new vendor without a unique TIN. The button will remain inactive until a TIN is added and the Search button clicked. If the search finds existing vendors with that TIN, the Add as New Vendor button will remain locked. If not, it will unlock
  3. Cancel - Press the SPACEBAR or single-click the cancel button to close the Vendor Validation Dialog without adding a new vendor.
