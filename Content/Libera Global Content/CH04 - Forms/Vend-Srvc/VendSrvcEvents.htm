Vendor Service Events


Events section.

 

Overview

Services associated with a vendor via the form cannot be used in the system until they are both activated and approved. To prevent a service associated with a vendor from being used you may either deactivate or un-approve the form.

 

Summary

Text appears notifying you if the form is activated and/or approved or not. When it is activated and approved, the text is green. When it isn't activated and approved, the text is red and tells you how to activate and/or approve the form.

 

 True for All Tabs

Selecting a record in the grid shows the values in the fields below the grid. You can add a Start Date and Comments in the fields provided. As records are added to the grid the number displayed in the tab changes to reflect the number of events.

 

 Approved

Shows a grid of Approval events. When approving a form, select the Action Row where it says, "Please click here to add a new record..." Doing so adds a record to the grid with today's date in the Added On column.

 

 Budget

This section is provided for informational purposes only. Data entered has no fiscal functionality.

  • Period No. - Type the number of payment periods.
  • Budget Amount – Type the dollar amount allocated for the service.

 

 Contract Amendment

Contract amendments do not replace previous iterations of that contract. A contract or contract amendment is still valid during the date range of that event even if there is a subsequent contract amendment.

For example, there is a contract running from 01/01/20xx to 03/31/20xx and there is an amendment that extends that contract from 04/01/20xx to 06/30/20xx. The user writes an authorization and enters a Service Start Date in February. This will use the original contract whereas a date in May would use the amendment.

 

 Contract Reactivate

To reactivate a previously terminated contract for the specified vendor’s service, complete the following fields.

  • Contract No. – Press ALT+F1 to open the lookup list. Use the ARROW keys to scroll through the options. Press ENTER to select an option. Alternatively, you may click the blue pull down arrow and then double-click an option in the list to select.

 

 Contract Terminated

To terminate a contract select the Action Row, where it says "Please click here to add a new record..." This starts a new record with the fields showing below the Contract Terminated Events grid. In the Contract No. field select the contract to be terminated.

  • Terminating a contract prevents users from authorizing the contracted service.
  • Terminated contracts appear in the View Contracts by Vendor PDQ.

 

 Current Rates

Rates added on the Current Rates event tab appear in the Current Rates from Events section of the form. They add to and/or override any rates that populated the section from the Service Information form. They are specific to this Vendor / Service combination and do not change the Current Rates on the form.

 

 Deactivate

Use the Deactivate event to deactivate the service for this vendor. The form must be both approved and activated before it can be used. Deactivating the form prevents the vendor/service combination from being planned or authorized.

 

 New Contract - Long Form

To add a new contract for the specified vendor’s service, complete the following fields.

  • Vendor has approved providing service for no payment when contract line item has been exhausted? - When set to Y (Yes), users are allowed to continue services through a CRP even if the contract has run out. This allows users to refer and continue moving a through the step process or approve s in the stand alone process if the contract amount has been met. A Payment Approval is not created when this scenario happens.
  • Pending Contract Amount - Use to add a pending amount for authorized contracts that are awaiting an increase in funds. Payments against this pending amount are queued in the Services Provided Ready for Payment PDQ and paid once the increase in funds is authorized. Payments awaiting the pending amount are highlighted yellow in the PDQ.
  • Counselor Approval Required for Payment - Used in the Community Rehabilitation Program/Provider (CRP) process. When a service is paid by Progress Note or Milestone and the CRP selects the outcome, if this field is selected, the counselor must approve the outcome before it can be paid.
  • Pay Set Fee Per Completed Service on Progress Reports - Used in the Community Rehabilitation Program/Providers (CRP) process. When selected service is paid by the set fee regardless of how many hours are applied to the service.

 

 New Contract - Short Form

Similar to the New Contract - Long Form section but with less fields.

 

 Unapproved

Use the Unapproved event to unapprove the service for this vendor. The form must be both approved and activated before it can be used. Unapproving the form prevents the vendor/service combination from being planned or authorized.
