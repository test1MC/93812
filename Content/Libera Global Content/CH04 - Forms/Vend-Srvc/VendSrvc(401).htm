Form


Top section of the form.

 

Overview

The form is used to link services to vendors. It shows details specific to the vendor service relationship. The information entered on this form serves as source data for numerous other forms related to the vendor.

 

 

Basic Information
  • The Name, Street, and Phone fields at the top of the form come in from the form.
  • You may select the Complete? field when all data entry on this form is completed. It's not required.

 

 Supplied Service

This section shows information related to the Supplied Service. For each form, only one service is selected. The following conditions apply to the Supplied Service field:

  • If you change the Supplied Service, all pre-existing events are deleted.
  • If a pricing tier is associated with the selected supplied service, the Vendor Service Events section populates with the existing rate information from the Service Information form.
  • If certification(s) are required for the service selected and the valid certification(s) are not identified on the form, a message appears stating that required certification(s) are invalid or missing: The service cannot be added due to the following reason(s): [Certification Name] Certification is missing.
  • Supplied Service – When a service is selected a brief description of the service appears in the bottom half of the Service Name lookup list window. Press CTRL+SHIFT+D to move cursor focus to the service description. Press CTRL+SHIFT+D again to return to the option list.
  • – If filled in on the form this field will automatically populate with selection of Supplied Service. If not, then type the appropriate service code. The ability to edit this field varies by agency.

 

 Vendor Service Events


Events section.

 

The Vendor Service Events sections is divided into multiple tabbed headings. The tabs vary on how the form is configured for your Agency. Regardless of the 'event' the tab functionality is the same. Please refer to Vendor Service Events for functional descriptions of each Tab.

 

 Current Rates from Events

This section is read-only, and shows the current rates either from the selected record of the Current Rates tab within the Vendor Service Events section or from the Service Information form.

To edit this data:

  • If it came from the form, you must edit it on that form.
  • If it came from the Current Rates tab, that's where you edit it.
  • Progress Notes select and use the Maximum Rate for the Pricing Tier or Current Rate if a Normal Rate is not specified. If the Current Rate has neither a Normal nor a Maximum Rate specified, the user is allowed to enter a desired amount.
  • Similarly, the number of hours provided per the Progress Note checks and makes sure the number doesn't exceed the Maximum Unit specified on the form.
  • Plans use the Current Rates from Events data. If data appears in this section it appears in Authorization and Plan forms. It can be changed in these forms but cannot go over any stated Maximum Unit or Rate.
  • Current Rates from Events data is only used if there are no Pricing Tiers or Contracts for this vendor/service combination.

 

 Miscellaneous Configuration
  • When the Is this a similar / comparable benefit? field is selected, if the Only show Similar benefit providers in the vendor lookup on authorizations* application setting is set to True, the Inventory form will only show services where this field is selected. *Libera manages this application setting.
  • When you first create the form, it pulls in the 1099 Reportable value from the form.
  • Selecting the Require Contract to Authorize? field prevents users from entering a Service Begin Date where no contract is active for the date entered. If it's not selected, users will be able to authorize the service as a non-contracted service after a contract has expired.
  • Selecting the Require Payment via Progress Notes? field prevents users from entering Payment Approvals manually for that vendor's service. Instead they will only be able to pay for services through progress reports and the invoice/payments generated when outcomes are approved.

 

 Service Types

Service Types come over from the Service Information form. They must match the Service Types on the form.

If you try to fill in a blank Service Type, the following message appears: The Service Type cannot be filled in as the Service Type on the Service Information form is blank.

If you try to change a Service Type, the following message appears: The Service Type can only be cleared or set to match the Service Type (8) on the Service Information form.

Service Type changes should be made on the form.

 

 Program Configuration

Note: This section is optional. If this section is not filled in, the system will consider the service to be allowed in any status for all programs.


Program Configuration section.

 

Use the Program Configuration section to configure the service for one or more programs. Refer to Multiple-Section Sections in Forms for more information.

  • The Program field links the other fields in the section to the selected program. You must select a program before you are allowed to fill in the other fields in the section.
  • Account Identifier / Take Money only from this Account Identifier - Allows you to link this service to one account identifier which controls the budget account selected when authorizing this service. When the Take Money only from this Account Identifier? field is selected, authorizations will ignore any accounts that do not have the selected Account Identifier. When the field isn't selected, the system prioritizes accounts with that Account Identifier first but, if none are available, account selection rolls over into other accounts.
  • Unplanned / Planned Statuses - These fields allow you to control when this service can be authorized based on the case status as of the Date of Authorization, on the Authorization form, for the selected program.
    • Unplanned Statuses - Controls when you can create a Diagnostic or Pre-Plan Authorization for this service. For example:
      • If you leave this field blank, you can create a Diagnostic Authorization in any status. Please note however, that Diagnostic Authorization forms have minimum and maximum status requirements that will prevent access to the form unless the case status is within that range.
      • Entering any status into the field allows you to only create a Diagnostic Authorization when the case is in the specified status.
        • If you enter only "unplanned" statuses, then you can only create a Diagnostic Authorization when the case is not in a planned status.
        • You can enter "planned" statuses to allow creation of a Diagnostic Authorization when the case is in the specified planned status.
      • You can write the word "none" to prevent the creation of Diagnostic Authorizations for this service all together.
    • Planned Statuses - Controls when you can create a Planned Authorization for this service. For example:
      • Leaving the field blank allows you to create Planned Authorizations in any status. Please note however, that Planned Authorization forms have minimum and maximum status requirements that will prevent access to the form until the case status is within that range.
      • Entering any status into the field allows you to only create a Planned Authorization when the case is in the specified status.
      • You can write the word "none" to prevent the creation of Planned Authorizations for this service all together.

Note:

  • The status rules only apply to the selected program. If no status rules are selected, then all statuses will be allowed by default.
  • If this section is populated from the Service Information form, you can edit it on the Vendor's Services form. Your changes are specific to the Vendor's Services form; they do not flow back to the Service Information form.

Authorizations to Clients

Authorizations to clients are treated a little differently because whether the Authorization to Client should be considered a planned authorization or not is dependent upon the case status as of the Date of Authorization on the Authorization form. If the authorization is for an unplanned status, it treats it as an unplanned authorization, otherwise, it treats it as a planned authorization. For example for a VR case, it means an authorization written for status 10 would use Unplanned Statuses and one written for status 12 would use Planned Statuses.

 

 Discounts

Enter information related to discounts into the fields in this section. This data doesn't affect Authorizations or Payments, it's for informational purposes only. You can add as many discount sections as necessary.

 

 Service Settings

You can add as many Service Settings sections as necessary.

  • Selecting a Service Setting automatically populates the Service Setting Description.
  • Once a Service Setting is selected, you may select one or more Service Areas.
  • The system can be configured to restrict services by county based on the Service Area selected. If this is true, then the service is only available on Authorizations if the county of the 's main residence matches one of the counties in the Service Area. Contact Libera Client Services for more information on this functionality.

 

 Service Delivery Starts

This section shows details about the delivery of the service.

 

 Additional Form(s) to Include

This section allows you to automatically launch additional forms when this service is selected. You can add as many additional form(s) to include sections as necessary.

  • On Which Form - Select a form that you want to apply an action to. Your selection will cause the Action Statement to update.
  • Action - Select the action that you want to occur on the form chosen in the previous On Which Form field. Your selection will cause the Action Statement to update.
  • Form to be included when this service is selected - Select the form you wish to be added or launched when this vendor service combination is selected. Your selection will cause the Action Statement to update.
  • Program - If a program is selected, the Action for the form selected in the On Which Form field will only happen if the form matches the program selected.
  • [Action Statement] – Automatically populated with selection of Action and Form to be included when this service is selected.
