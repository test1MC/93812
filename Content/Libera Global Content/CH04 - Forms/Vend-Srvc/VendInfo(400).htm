Form

Overview

The form collects demographic information about the vendor. The information entered on this form serves as source data for numerous other forms related to the vendor.

 

Accessing the Vendor Information Form

  • Access to this form is limited by your User Role.
  • Vendor data must be displayed in the Data Tree before you can add a new vendor or update an existing service.
  • You may select the Vendors option from the Application Home Page or the View All Vendors PDQ from the PDQs tab of the File Drawer.
  • Run the View All Vendors PDQ.
  • To add a new vendor, select the Search / Add New Vendor button at the bottom of the Data Tree to launch the Vendor Validation Dialog.
  • To view an existing Vendor Information form select the vendor in the Data Tree.

 

 

Form Fields
  • Vendor Legal Name - This is the vendor's legal name as it was entered into the Vendor Validation Dialog. This is the name that appears in grids and forms.
  • Search By - The Vendor Validation Dialog uses the text entered into this field when searching for existing/duplicate vendors (in addition to Vendor Legal Name, Doing Business As Name, First Name, and Last Name). It also appears in the View All Vendors PDQ in the column of the same name.
  • Print Legal Name during Mail Merge? - When selected the Vendor Legal Name is used when mail merging.
  • Doing Business As Name - (DBA). The Vendor Validation Dialog uses the text entered into this field when searching for existing/duplicate vendors (in addition to Vendor Legal Name, Search by, First Name, and Last Name). It also appears in the View All Vendors PDQ in the column of the same name.
  • Store Number - Use if the vendor has a store number.
  • Title - This is the honorific title of the person in the First and Last Name field. For example: 'Mr.', 'Mrs.', 'Miss', etc.
  • First Name - The Vendor Validation Dialog uses the text entered into this field when searching for existing/duplicate vendors (in addition to Vendor Legal Name, Doing Business As Name, Search By, and Last Name). It also appears in the View All Vendors PDQ in the column of the same name.
  • Last Name - The Vendor Validation Dialog uses the text entered into this field when searching for existing/duplicate vendors (in addition to Vendor Legal Name, Doing Business As Name, Search By, and First Name). It also appears in the View All Vendors PDQ in the column of the same name.
  • Tax ID Number* - Tax payer Identification Number (TIN). This is the identification number used by the Internal Revenue Service (IRS). A TIN may be the Social Security Number of the sole proprietor or the Employer Identification Number (EIN). You can add a new vendor without a TIN however, you will not be able to approve the vendor without one.
  • Various Agency Specific Fields - There are variety of fields that may appear following the Tax ID Number field. These fields are related to your Agency's fiscal system and are required for successful batch processing.
  • Pay to Vendor – Optional. Use this field if payments and/or other communications go through another vendor. Make a selection from the lookup. Selection fills in number and description of the selected vendor.
  • Hide from Authorization? - When selected, this vendor is not available to be added to authorizations.
  • Complete? - Optional field. Select when data entry is completed.

 

 Vendor Events



Events section.

 

The Vendor Events section is divided into tabbed headings.

The Vendor Events section is divided into sections.

  1. Summary - Text appears notifying you if the Vendor is activated or not. When it is activated, the text is green. When it isn't activated, the text is red and tells you how to activate the service.
  2. Activate/Reactivate (#) - Shows a grid of activate/reactivate events. When activating a Vendor, select the Action Row where it says, "Please click here to add a new record..." Doing so adds a record to the grid with today's date in the Added On column.
  3. Deactivate (#) - Shows a grid of Deactivate events. When deactivating a Vendor, select the Action Row where it says, "Please click here to add a new record..." Doing so adds a record to the grid with today's date in the Added On column.

True for all Vendor Events Tabs



Activate/Reactivate selected.

 

Selecting a record in the grid shows the values in fields below the grid. You can add a Start Date and Comments in the fields provided. As records are added to the grid the number displayed in the tab changes to reflect the number of events.

 

 Required Certifications


Vendor Certifications section.

 

  • Vendor Certifications - Optional. Your Agency may require Vendors to maintain one or more certifications. Use the lookup field to select the required certification. Your Agency maintains the list of certifications. Refer to Multiple-Section Sections in Forms for more information.

 

 Vendor Business Address
  • Zip+4* – When selected, the City*, State, and County fields automatically populate. Enter the 4-digit Zip Code extension if known, it's OK to leave the extension field blank.
  • Country – When adding vendor as a new vendor this field is automatically populated with USA. When adding vendor as a new location to an existing vendor this field is copied from the existing vendor.
