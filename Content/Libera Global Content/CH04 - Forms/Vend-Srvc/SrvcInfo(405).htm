Service Information Form

Overview

The Service Information form collects information about the service. The information collected on this form serves as source data for numerous other forms and functionality related to the service throughout System 7™. The Service Information form can be opened directly from the Data Tree of the PDQ.

Accessing the Service Information Form

  • Access to this form is limited by your User Role.
  • Service data must be displayed in the Data Tree before you can add a new service or update an existing service. You may select the Services option from the Application Home Page or the View All Services PDQ from the PDQs tab of the File Drawer.
  • To add a new service, select the Add New Service button at the bottom of the Data Tree to launch a new Service Information form.
  • To view an existing Service Information form select the service in the Data Tree.

 

Form Sections

 

Basic Information


Example of the Basic Information section.

 

  • Parent Service - If this service is a child/sub service, use this field to link it to its Parent Service. For additional information refer to Parent Child/Sub Services.
  • Commodity Type - Only services that are classified as a "Good" can be transferred in equipment transfer forms (if used by your Agency).
  • Complete? - Is the Service Information form completely filled out?

 

 Child/Sub Services

If this service is a Parent service, this section lists all the child/sub services linked to it. You may link additional child/sub services. Refer to Multiple-Section Sections in Forms for more information.

 

 Service Events


Events section.

 

The Service Events section is divided into three tabbed headings.

  1. Summary - Text appears notifying you if the Service is activated or not. When it is activated, the text is green. When it isn't activated, the text is red and tells you how to activate the service.
  2. Activate/Reactivate (#) - Shows a grid of activate/reactivate events. When activating a Service, select the Action Row where it says, "Please click here to add a new record..." Doing so adds a record to the grid with today's date in the Added On column.
  3. Deactivate (#) - Shows a grid of Deactivate events. When deactivating a Service, select the Action Row where it says, "Please click here to add a new record..." Doing so adds a record to the grid with today's date in the Added On column.

True for all Service Events Tabs


Activate/Reactivate tab selected.

 

Selecting a record in the grid shows the values in fields below the grid. You can add a Start Date and Comments in the fields provided. As records are added to the grid the number displayed in the tab changes to reflect the number of events.

 

 Required Certifications


Required Certifications section.

 

  • Required Certifications - Optional. Your Agency may require Vendors who provide this service to maintain one or more certifications. Use the lookup field to select the required certification. Your Agency maintains the list of certifications. Refer to Multiple-Section Sections in Forms for more information.

 

 Service Types


Service Types section.

 

Service Types are used to categorize the service. They are also used for reporting purposes. They are also used to control when a service can be used. All services must have a Service Type for at least one program before you are allowed to activate the service. The lookups for each program are determined either by the Federal government or your Agency.

 

 Miscellaneous Codes

There can be many different codes that are specific to your Agency. One or more of these codes may be required to work with the fiscal process for your Agency. If a miscellaneous code field is not filled in, it could result in a payment for this service not being processed for payment. If in doubt, check with your Fiscal staff.

 

 Rate Information


Rate Information section.

 

When this section is filled in the information provided flows into the Vendor's Services form. These values can be overridden on the Vendor's Services form if configured by designated staff to do so.

When editing the Rate Information section after the service is linked to one or more vendors, you are prompted to select which vendor services you wish to update with the new value. This happens for each field you update. You may select multiple records in the grid that appears. Refer to Multiple-Section Sections in Forms for more information.

 

 Pricing Tiers Program Configuration

Note: This section is optional. If this section is not filled in, the system will consider the service to be allowed in any status for all programs.


Program Configuration section.

 

Use the Program Configuration section to configure the service for one or more programs. Refer to Multiple-Section Sections in Forms for more information.

  • The Program field links the other fields in the section to the selected program. You must select a program before you are allowed to fill in the other fields in the section.
  • Account Identifier / Take Money only from this Account Identifier - Allows you to link this service to one account identifier which controls the budget account selected when authorizing this service. When the Take Money only from this Account Identifier? field is selected, authorizations will ignore any accounts that do not have the selected Account Identifier. When the field isn't selected, the system prioritizes accounts with that Account Identifier first but, if none are available, account selection rolls over into other accounts.
  • Unplanned / Planned Statuses - These fields allow you to control when this service can be authorized based on the case status as of the Date of Authorization, on the Authorization form, for the selected program.
    • Unplanned Statuses - Controls when you can create a Diagnostic or Pre-Plan Authorization for this service. For example:
      • If you leave this field blank, you can create a Diagnostic Authorization in any status. Please note however, that Diagnostic Authorization forms have minimum and maximum status requirements that will prevent access to the form unless the case status is within that range.
      • Entering any status into the field allows you to only create a Diagnostic Authorization when the case is in the specified status.
        • If you enter only "unplanned" statuses, then you can only create a Diagnostic Authorization when the case is not in a planned status.
        • You can enter "planned" statuses to allow creation of a Diagnostic Authorization when the case is in the specified planned status.
      • You can write the word "none" to prevent the creation of Diagnostic Authorizations for this service all together.
    • Planned Statuses - Controls when you can create a Planned Authorization for this service. For example:
      • Leaving the field blank allows you to create Planned Authorizations in any status. Please note however, that Planned Authorization forms have minimum and maximum status requirements that will prevent access to the form until the case status is within that range.
      • Entering any status into the field allows you to only create a Planned Authorization when the case is in the specified status.
      • You can write the word "none" to prevent the creation of Planned Authorizations for this service all together.

Note:

  • The status rules only apply to the selected program. If no status rules are selected, then all statuses will be allowed by default.
  • If this section is populated from the Service Information form, you can edit it on the Vendor's Services form. Your changes are specific to the Vendor's Services form; they do not flow back to the Service Information form.

Authorizations to Clients

Authorizations to clients are treated a little differently because whether the Authorization to Client should be considered a planned authorization or not is dependent upon the case status as of the Date of Authorization on the Authorization form. If the authorization is for an unplanned status, it treats it as an unplanned authorization, otherwise, it treats it as a planned authorization. For example for a VR case, it means an authorization written for status 10 would use Unplanned Statuses and one written for status 12 would use Planned Statuses.

 

 Plan Configuration


Plan Configuration section.

 

Use the Plan Configuration section to configure the service for Plans.

  • Hide from plan lookup? - When marked Y (Yes), users will not be able to add this service to Plans. For example, if this is a child/sub service, you would set this to Y (Yes).
  • Requires Supervisory Signature on Plan? - If marked Y (Yes), a signature line is added to the Plan when this service is selected. The supervisory signature line appears after the first signature is dated on the Plan. The required signature is for the immediate Supervisor of the counselor who signed the Plan.
  • Create a Plan Amendment to be Resigned? - When you update an existing plan it causes a Plan Amendment. Your Agency can require the Plan Amendment to be signed or not. Regardless of how your Agency has configured this process, if the response to this field is Y (Yes), users will be required to sign the Plan Amendment.
  • Contingent on Economic Need? – Selecting Y (Yes) requires the client to have the Economic Status Report form on file indicating economic/financial need.
  • Plan ACL - Allows you to control which groups of users can add this service to Plans based on User Role. You may type a list of User Roles, for example: 23,24,123,1001,15071, or you may use the Build Plan ACL button to select which roles to include or exclude. When you use the Build Plan ACL button, the Build Plan ACL section becomes visible. When you select either the Include Roles or Exclude Roles link, a lookup opens displaying your Agency's User Roles. You may select one or more User Roles using the check box column in the grid. Select the Save button to save your changes. This closes the Build Plan ACL section and displays a comma separated list of ACLs in the Plan ACL field.
  • If Service exceeds... - Allows you to control approval requirements based on User Role or Employee for the dollar amount indicated. You may enter a dollar amount, which then will unlock the by Role and by Employee fields. The lookups for these fields will only allow one choice. If you mark the Include All Signatures up to and including this role? as Y (Yes), additional signature lines appear for all of the supervisors between the user and the role indicated.

 

 Authorization Configuration


Authorization Configuration section.

 

Use the Authorization Configuration section to configure the service for authorizations.

  • Hide from authorization lookup? - When this is set to Y (Yes), the service will not show in an authorization lookup unless it is an Agency Authorization and the Show on Office Authorizations field is marked Y (Yes). If this is a Parent Service, you would set this to Y (Yes).
  • Show on Office Authorizations? - When marked Y (Yes), this service appears in the service lookup for the Agency Authorization form. When marked N (No) or not set, it will not appear in the service lookup for the Agency Authorization form.
  • Print cost as TBD? - When marked Y (Yes), when you print the Authorization, the Unit Price and Total Cost will print as "TBD" (To Be Determined).
  • Requires Terms and Conditions Agreement? - The service cannot be authorized unless the Vendor has a Terms & Conditions Event on the Vendor's Services form that has not expired. Related Application Setting: Number of days after which a vendor's Terms & Conditions agreement is no longer current .
  • A detailed description of the services to be provided must be included? - When marked Y (Yes), the Service Detail field on the line item is required when authorizing this service.
  • Requires Exemption Form on Authorization? - Setting to Y (Yes) causes the Exemption button to appear on authorization forms.
  • Allowed to be authorized as a Service to Groups? – When marked Y (Yes), allows service to be used on group authorizations.
  • Request For Quotes? / Starting at What Amount? - Setting to Y (Yes) causes the button to appear on authorization forms when the service amount exceeds the amount entered. This button opens the form.
  • Price Confirmation Request? / Starting at What Amount? - Setting this to Y (Yes) will cause the Price Confirmation Request button to appear on authorization forms when the service amount exceeds the amount entered. This button opens the Price Confirmation Request form.
  • Is this a taxable item? - When marked Y (Yes), sales tax is calculated for this service at the time of authorization.
  • Authorize ACL - Allows you to control which groups of users can add this service to authorizations based on User Role. You may type a list of User Roles, for example: 23,24,123,1001,15071, or you may use the Build Authorize ACL button to select which roles to include or exclude. When you use the Build Authorize ACL button, the Build Authorize ACL section becomes visible. When you select either the Include Roles or Exclude Roles link, a lookup opens displaying your Agency's User Roles. You may select one or more User Roles using the check box column in the grid. Select the Save button to save your changes. This closes the Build Authorize ACL section and displays a comma separated list of ACLs in the Build Authorize ACL field.
  • If Service exceeds... - Allows you to control approval requirements based on User Role or Employee for the dollar amount indicated. You may enter a dollar amount, which then will unlock the by Role and by Employee fields. The lookups for these fields will only allow one choice. If you mark the Include All Signatures up to and including this role? as Y (Yes), additional signature lines appear for all of the supervisors between the user and the role indicated.

 

 Shared Configuration


Shared Configuration section.

 

Use the Shared Configuration section to configure the service.

  • Supports Small Business Enterprises? - Affects the RSA-2 (Schedule II Line 13). Please note that this may be phased out due to the 2014 regulation changes.
  • Text entered into the Brief Service Description field appears on authorizations and plans as configured by your Agency.
    Related Application Settings:
    • Determines whether the Service Brief Description field is shown in the Item lookup on the Meyer Center Request and Transfer forms
    • Populate Service Details on Authorization with Brief Service Description on from Service Information
    • Show Brief Service Description on Service form
    • Show the Brief Service Description on Plans
  • Number of Times Provided Per Case – Type the maximum number of times this service may be provided per case.
  • Amount Allocated Per Case – Type the maximum dollar amount allowed for this service per case.
  • Number of Times Provided Per Client – Type the maximum number of times this service may be provided for one client across all cases.
  • Amount Allocated Per Client – Type the maximum dollar amount allowed to be allocated per client across all cases.

Additional Signature Controls in the Shared Configuration Section

You may specify signature rules on a per service basis. This is applied on the Service Information form, via the fields following this statement:

The following fields control additional signatures based on role for the life of a case. Signature Role and (Min or Max Amount) fields are required.

  • The Value Type field controls the field labels for Min Quantity and Max Quantity. For example:
    • Setting the Value Type to Total the field labels change to Min Amount and Max Amount and opens them for editing.
    • Setting the Value Type to Unit Price the field labels change to Min Amount and Max Amount opens them for editing, and changes the Duration to Instance.
    • Setting the Value Type to Quantity doesn't change the field labels, but opens them for editing.
  • The rules are applied when users select the service for Authorization, Plan, or Internal Referral forms. You designate when you want the rules applied by responding Y (Yes) to the Auths?, Plans?, and/or Internal Referrals? fields.
  • The Signature Role field controls which additional roles need to sign the form. The Min Quantity/Amount field states the minimum quantity to trigger the additional signatures to be added to the form. The Min and Max Quantity/Amount fields set a cost range for which to add the additional signature line.
  • The additional fields relating to the projected quantity and cost are not required fields. When the calculations are not required the plan can be signed without these fields being populated.
  • When you set Duration to Form, instead of calculating the total of services for the entire life of the case or for a certain time frame, the system calculates the total of the services on the current form.
  • You may add as many additional signature rules as needed. Please refer to Multiple-Section Sections in Forms for more information.

 

 Additional Form(s) Text

This section allows you to automatically add text to forms that access this service. You can add as many additional form(s) text sections as necessary. Please refer to Multiple-Section Sections in Forms for more information.

  • On Which Form - Select a form that you want to apply an action to. Your selection will cause the Action Statement to update.
  • Short Description – This lookup list is maintained by designated Agency employees via the Dictionary Administration table: LVR_Service_Authorization_Text.
  • Text to be included on form when service is selected – When you select the Short Description, the text from the Long Text column in the Short Description lookup fills into this field.

 

 Additional Form(s) to Include

This section allows you to automatically launch additional forms when this service is selected. You can add as many additional form(s) to include sections as necessary. Please refer to Multiple-Section Sections in Forms for more information.

  • On Which Form - Select a form that you want to apply an action to. Your selection will cause the Action Statement to update.
  • Action - Select the action that you want to occur on the form chosen in the previous On Which Form field. Your selection will cause the Action Statement to update.
  • Form to be included when this service is selected - Select the form you wish to be added or launched when this Vendor's Services is selected. Your selection will cause the Action Statement to update.
  • Program - If a program is selected, the Action for the form selected in the On Which Form field will only happen if the form matches the program selected.
  • [Action Statement] – Automatically populated with selection of Action and Form to be included when this service is selected. You may not edit this field.
