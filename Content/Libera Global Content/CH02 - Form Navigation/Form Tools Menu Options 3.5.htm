Form Tools Menu Options


Tools menu.

 

  • Check Spelling - Opens Spell Check dialog window. Hotkey: F7.

Spell Check Dialog Window


Spell Check window.

  • Word Not in Dictionary - Displays misspelled/unidentified word in red.
  • Change To - Displays selected suggestion. Alternatively you may type the correct spelling.
  • Suggestions - List of suggested correctly spelled words.

Press SPACEBAR or click to activate the following buttons.

  • Ignore - Ignores misspelled word. Hotkey: ALT+I (i).
  • Ignore All - Ignores all instances of misspelled word. Hotkey: ALT+A.
  • Add - Adds word to spell check dictionary. Hotkey: ALT+A.
  • Replace - Replaces misspelled word with Change To word. Hotkey: ALT+R.
  • Replace All - Replaces all instances of misspelled word with Change To word. Hotkey: ALT+E.
  • Cancel - Closes Spell Check dialog window. Hotkey: ALT+C.
  • Take Snapshot - Makes a copy of the form and places it in a Case Note.
  • Form Change History - Opens the Data Audit Change History Tool in a new window. Please note that not all users will have access to this option. Access to these tools is controlled by your System Administrator.

Form Change History Window


Form Change History window.

  • Date/Time - Actions are listed with the most recent at the top. Dates and times are displayed in the form MM/DD/YYYY HH:MM:SS AM/PM.
  • Table and Column - These list the data table and the column within it changed by the action. If the action was editing a field, then the Column may likely be similar to the field name.
  • Old Value and Value - These are the values in the data table (and, in some cases, the form field) before and after the action was taken. The Old Value column is most useful for restoring lost data.
  • Action - Like Flags in the Data Watch Audit Log, this is a simple description of the action taken. Possible values are "Updated" for changing the data in a field and "User Activity" for loading and viewing forms.
  • Primary Key and Primary Key# - A primary key can be a number or a string of text. Whichever type of primary key is used for this action, it will be recorded in the appropriate column.
  • Sometimes the Primary Key may not always be the same as the primary key of the data table, but some other useful information, such as the user name used to log on.
  • User Name - This is the name of the user that performed the action.
  • Field Change History - Opens the Change History for [field name] in a new window. Please note that not all users will have access to this option. Access to these tools is controlled by your System Administrator. This only lists information about actions performed on the selected field (field with cursor focus). If it is selected without the cursor focus in any field, then nothing will happen. Field Change History is not available for all fields in a form. If not available for the field you have selected, a message will appear recommending that users utilize the Form Change History option.

Change History Window


Change History for [field name] window.

  • Date/Time - Just like the Data Watch Audit Log, actions are listed with the most recent at the top. Dates and times are displayed in the form MM/DD/YYYY HH:MM:SS AM/PM.
  • Old Value and Value - These are the values in the data table (and, in some cases, the form field) before and after the action was taken. The Old Value column is most useful for restoring lost data.
  • Action - Like Flags in the Data Watch Audit Log, this is a simple description of the action taken.
  • Form - This lists the name of the form the field is in followed by the date that this particular instance of the form was created.
  • User Name - This is the name of the user that performed the action.
