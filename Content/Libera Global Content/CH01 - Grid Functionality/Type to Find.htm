Type to Find

Type to Find allows you to quickly jump to a record (cell) by starting to type text (alphanumeric characters) when cursor focus is in the grid. Alphanumeric characters are A-Z, 0-9, and the number pad digits. Use the Filter Row to search for special characters such as an ampersand (&) or pound sign (#).

Search Behavior

  • Searches the column from the current cursor location to the last loaded record, then loops back to the top of the table.
  • Searches by the cell values starting with the typed alphanumeric characters. For example, as you type "tes", it first jumps to a cell starting with t, then tries to find one starting with "te", then one starting with "test", if one exists.
  • If you pause too long between characters (more than one second), the search assumes you wanted to start over and resets the search string. For example, if you type "t (pause) e (pause) s", it first jumps to a cell starting with t, then one starting with e, then one starting with s.
  • Repeatedly pressing the same character, for example "zz" or "zzz" etc., the search assumes you want to jump between cells beginning with the letter z, not the word "zzz".
  • If you mistype a character in a search string, pause a moment and begin typing again.
  • Search check box columns by typing Y, 1, X, or T to jump between selected cells. Type N, 0, or F to jump between clear cells.
  • You can use the Filter Row and Type to Find together. This is especially helpful if you're working with a grid that has hundreds of records.
  • If a cell value contains punctuation or symbols intermixed with alphanumeric characters, just type the alphanumeric characters to find a cell. For example, if a cell value is "03/06/2015", just start typing "0306" etc. to find the date. Don't type the slashes or other punctuation. The grid searches without the punctuations.

Grid Structures Supported

When grids are grouped by one or more columns, the search goes through all groupings. It automatically expands collapsed groupings when it finds a matching cell.

When searching within a grid that has parent and child tables such as the Vendor/Service List, the search stays within the table that has focus. For example, if cursor focus is in the parent table, searching jumps from parent table record to parent table record. If cursor focus is in a child table, it stays within the child table (it doesn't jump from child table to child table).

Note: If you are in an editable cell/column, typing an alphanumeric character starts editing the cell. It will NOT start a Type to Find search. For example when you're setting up user preferences or if you're working in the System Administration module.

Limitations

Type to Find only searches loaded records. To search all records in a PDQ, use the Filter Row.

 

  • When searching within the Data Tree, performance will be slower if your user preference for Open electronic case folder automatically? is Yes. When it's set to yes, the system attempts to load the ECF as cursor focus jumps from cell to cell within the column. If this is problematic for you, consider changing the setting from Yes to No.
