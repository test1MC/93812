Account List PDQ.

 

Overview

The PDQ displays all accounts used to fund services. This information includes account identification, financial status, and dates of activity. Data relevant to any external accounting is also displayed. You will only see accounts for the offices associated with the caseloads or teams which you are linked (if your Agency uses caseloads and/or teams), and the office which appears on your Employee Information form. Other user roles or settings may affect the PDQ results. If you have any questions about the data you are seeing, please contact your System Administrator.

Access the PDQ one of two ways:

  • Select the Accounts option under Resource Management on the Application Home Page.
  • Open the File Drawer to the PDQs Tab and filter for Account List. Select the PDQ in the resulting list to run or open it.

 

PDQ Results
  • Selecting an account record in the Data Tree opens the form.
  • Use the Create Account button to add new top level or accounts to the system. Selecting the button opens a blank form ready to be filled out.
  • Select an Upper Level Account record in the Data Tree, then use the Create button to add a new to that account. You cannot add s to Authorization accounts.

 

 Column Header Definitions

Note: As some of the column headers are self explanatory, not all are defined.

  • Account Number - This is a combination of the values from several fields on the form. It's explained in detail in the help content for the Account Information form.
  • Account Identifier Group - Displays the title of the Account Identity Group associated with the Account Identifier. It provides an additional way to categorize or group data in the grid.
  • Special Project - When authorizing services for s, authorizations will select accounts where the Special Project (on the form) matches the ’s Special Project (on the Application and Intake form) or the Default Project Description for the ’s program. Default Project Descriptions are set for programs in the Dictionary Table Administration module, specifically the Account Special Project Codes table. For more information on this, please refer to the help content for the Account Information form.
