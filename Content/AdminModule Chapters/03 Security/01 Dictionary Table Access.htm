Dictionary Table Access.

 

Edit Mode: Grid

Description: The Dictionary Table Access option allows Administrators to control which Dictionary Administration tables are visible and editable and to which user ID roles. This option is edited directly on the grid. The columns that Administrators can edit are Description, Read ACL, and Write ACL.

 

Column Definitions

  • Table - The table name within the data base. This is managed by Libera. Administrators may not change table names.
  • Description - Additional information provided to help you identify where or how the dictionary table is used.
  • Read ACL - Controls who can see the dictionary table within Dictionary Administration.
  • Write ACL - Controls who can edit the dictionary table within Dictionary Administration.

Modifying Dictionary Table Access

  1. Open (right-click or press F4) the cell context menu in the cell that you will be changing. Select the Edit otpion. This opens the cell up for editing.
  2. Change the contents of the cell as desired.

 

  • The ACL columns take a list of comma separated User Roles. There is no lookup for these fields, it's best if you have the User Roles on hand before editing.
  • Leaving an ACL cell blank means that all users with access to Dictionary Administration can see and/or edit the contents of the cell.
  • Placing an exclamation point (!) directly in front of a User Role means that users with that User Role cannot see and/or edit the Dictionary table.
  • You can allow non-System Administrators to edit Dictionary tables by giving them the Dictionary Administration role, 1100. This allows them to edit Dictionary tables via the Dictionary Administration PDQ.
  • The Read and Write ACLs do not have to be set identical. You can allow users to see Dictionary tables without giving them the ability to edit them.
