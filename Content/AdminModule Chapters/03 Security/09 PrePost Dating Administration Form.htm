Pre/Post Dating Administration Form

The Pre/Post Dating Administration form is divided into two pages:

  1. The rule section (first page)
  2. Pre/Post Test Page (second page)

 

Rule Section (first page)



The first page of the Pre Post Dating Administration form.

 

The first page is the information related to the rule. Each field corresponds to the column descriptions in the PDQ Results section above. Values entered into the fields appear in the grid. The form fields have the following rules (the same rules apply to the test page):

  • If the program field is not filled in, then Status From / In and Status are not editable and are blank.
  • If the program field is filled in, then Status From / In is editable.
  • When the Check Type is a non status change, Status To is not editable and is blank.
  • When the Check Type is All or Status Change, Status To is editable.
  • In order to enable the rule each of the following fields must be filled in:
    • Role
    • Order
    • Check Type
    • Maximum # of Days Ahead
    • Maximum # of Days Back
  • Leaving any of the # of days in Quarter X Grace Period fields blank results in the Maximum # of days ahead and Maximum # of days back rules being applied.
  • The form fields only display the configuration of the selected rule. The Pre / Post Dating Administration with Order PDQ is displayed within the form. It shows the current rules in the PDQ so you can see the other rule configurations. Any modifications to the selected rule won't show in the grid until the grid is refreshed.

 

Pre / Post Test Page (second page)



The second page of the Pre Post Dating Administration form.

 

This page allows you to test new rules to see the number of days back / forward / quarter a user or a list of roles will have when using the system.

  • User - You'll be testing the selected user's pre / post dating ability based on their roles and the Check Type you select.
  • List of Roles - This populates with the selection of the User. You can edit this field if desired. This allows you to test different scenarios for the user.
  • Check Type - This is where you determine if the test is for status changes, non status changes or both.
  • Program - Making a selection from the lookup allows you to test the rules for a specific program. Leaving it blank tests all of the global rules (rules not associated with a program).
  • Status From / In - If you're testing status changes, enter the status the case is currently in and will be moving from.
  • Status To - This is the status the case is moving to.
  • Date to check - This is the date you are either pre or post dating to. The rules always run for today's date.
  • Show Pre Post Dating Rules for Test - This button performs the test and returns the information as to what the number of days back / forward / quarter will be for both methods for the options selected in the test.
  • The Pre/Post Dating with Order section displays the rule that applies to the test criteria entered. The grid highlights the first rule found that meets the criteria (Rule ID will be filled in). If Rule ID is not filled in, then the system settings will be used. The specifics of the highlighted record appear in the fields below the section header.
  • The Pre/Post Dating by Maximum amount of Rules section can be hidden based on the application setting, Whether to show the Pre/Post Dating by Maximum amount of all Rules Section on the Pre Post Dating Form. This setting is set to true (on) by default and should be left on until after the ordering method is turned on and you are confident that the rules setup in the ordering are sufficiently tested.
