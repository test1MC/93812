Role Administration Form


Role Administration form.

 

The Role Administration form allows you to create and modify roles. All data entered on the form appears in the Role Administration grid as follows:

Form Fields

  • Number - this is the unique ID number of the role. When you add a new role to the system it must have a number higher than 1000 and be unique.
  • Short Description - This is the short description of the role. It must be 20 characters or less.
  • Long Description - This is the long description of the role. Use it to be more descriptive of the intent for the role. It must be 100 characters or less.
  • Program - Optional. You may associate the role with a specific program with the system. Select the program from the lookup. If the role is intended to be used across multiple programs, leave this field blank.
  • Roles That Can Assign - You may restrict who can assign this role to others if desired. This is commonly used when you have outside agencies working in your system.

 

Why Add a New Role?

Ideally, all necessary roles are created when the application is initially developed. However, changing agency needs may require additional roles. You can add new roles and modify existing roles in Role Administration located in the System Administration module.

For example, let's start with a basic program X counselor role (where X represents a particular program). All counselors that work with program X are assigned this role. Now let's assume that some counselors have additional duties that require they have access to additional functionality, forms, and so on within System 7™ that the other program X counselors do not need to access.

If you only want to give these special rights to some but not all program X counselors, you would create a new role that gives access to the additional functionality and then you would apply it only to the program X counselors that need it. This gives you finite control over what users have access to in System 7™.
