Primary Assignment Validation ECID 99000005

This edit check checks for primary assignments at status changes throughout the life of the case. This edit check is related to the Application Setting, "Require the Program field to be filled in for Primary client assignments". When set to true, then the Primary Assignment must have the primary assignment for that program for the date entered.

If you want to change the this application setting, please contact Libera.

 

Additional Configuration Information

Please note, you may review these settings but not change them. If you want them changed, please contact Libera.

 

Application Setting: A Comma Separated List of Allowed Types of Primary Assignments

This setting controls what assignment is required and is configurable by component. This setting stores a comma separated list of values. Possible values are 1, 2, or 3.

1 = Require Primary Team Assignment

2 = Require Primary Caseload Assignment

3 = Require Primary Worker Assignment

The 99999 extended system setting entry controls the primary assignment required when changing status. If the value of the system setting is set to a particular value, all other components will require that assignment based on the date that component was filed. Status 00 to 08 will not fire these checks.

 

Application Setting: Forms that can be opened when the client does not have a primary assignment

This setting controls what forms are allowed to be filed without a primary assignment.

Messages associated with this edit check:

  • This client has a primary assignment to [Counselor Title] Name on [MM/DD/YYYY].
  • This client has no primary assignment on [MM/DD/YYYY].
  • The [client must have (at least one of) the following assignment(s) on [MM/DD/YYYY]: [list of assignment types required]
