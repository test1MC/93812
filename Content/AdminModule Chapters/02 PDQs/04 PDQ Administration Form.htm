PDQ Administration Form


The top portion of the PDQ Administration form.

 

Overview

Use the PDQ Administration form to manage settings for the selected PDQ.

 

 

Form Functionality

  • PDQ Number - It's a unique identification number assigned to the PDQ. It appears in the PDQ XML ID column of the PDQ Administration grid. It's also known as the Component # of the PDQ. It's managed by Libera.
  • Name - This is the name of the PDQ. It appears in the Name column of the PDQ Administration grid and in the Name column of the PDQ grid displayed on the PDQs tab of the File Drawer.
  • Description - This is a brief description of the PDQ's functionality. It appears in the Description column in the PDQ grid displayed on the PDQs tab of the File Drawer.
  • Module - The PDQ loaded in the Desktop determines the module. The value in the Module field filters the other tabs in the File Drawer based on the PDQ loaded in the Desktop.
  • Flags - Essentially, this is a "tag" field for the developers. Any value in this field is informational only and doesn't impact PDQ functionality in any way. A common usage example is that it displays a parent PDQ, if any. This functionality is managed by Libera.
  • Arguments - Developer defined parameters that are passed into the PDQ. They run in the background, invisible to the end user. This functionality is managed by Libera.
  • Category - This controls where the PDQ appears in the PDQ list on the PDQs tab of the File Drawer.
  • Access ACL - Enter a comma separated list of User Roles that you want to have access to this PDQ or, use the Build Access ACL button to create the list.

 

PDQ Definition Options


PDQ Definition Options section of the PDQ Administration form.

 

This section allows you to change the order of the columns in PDQs to your Agency's preference. This is a new feature that already has enhancements planned to make it easier to use. If you have any questions about making changes in this section, please contact Libera. This is a multiple-section section which allows you to reorder columns in complex tables. A complex PDQ is one that has one or more levels of child tables such as the Vendor/Service list or almost any accounting PDQ. A simple PDQ is one that doesn't include one or more levels of child tables.

  • Table Name - Optional for simple PDQs, required for complex PDQs. At this time, if you're working with a complex table, please contact Libera for the table's name.
  • Column Order - Displays the PDQ's column names from the database (not what you see in the grid). You can reorder the columns in the PDQ by moving the column names around within this field.
  • Get Columns from PDQ - Use this button to populate the Column Order field with all of the PDQ's column names from the database (not what you see in the grid), even the columns not visible to your Agency. They populate the field in alphabetical order vs the order that they currently appear within the PDQ grid.
  • SQL Prefix and Suffix - Displays the code that is pre or post-appended to the SQL statement in the PDQ. These fields allow Libera to fine tune and improve PDQ performance without being required to make file system changes. Agencies will not be able to modify these fields.
  • Refresh PDQ Cache - Use this button to make your column order changes take effect.

 

  • Copy the contents of the Column Order field into a text file before making any changes. This way you can "undo" any changes you make.
  • You can drag and drop the names within the Column Order field by highlighting the name and using the mouse to drag it to another position in the list. Watch closely when you do this, you'll see the cursor focus move behind the text you're dragging. This indicates the location that the text will be dropped. It's important that it's dropped between column names and not in the middle of a column name.
