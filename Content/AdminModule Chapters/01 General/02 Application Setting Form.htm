Application Setting Form


Application Setting form.

Overview

The Application Setting form is used when editing configurable application settings for your Agency.

Prerequisites

Users must have System Administration rights to access this form.

Form Fields

  • The Value field is related to the title and description of the setting. If the application setting mentions a "...listing of forms...", "...list of roles...", "Calculate the OOS always", "Show X", or "...number of days..." the value is the response. For example if the setting is "Number of days before an attachment becomes locked" and the value is 10. 10 is the number of days.
  • Settings can also have Extended Settings. The use of Extended Settings allows you to apply different settings to different components.

Extended Settings

  • When extended settings are allowed for an application setting, you may add as many as necessary.
  • When a setting has extended settings do not assume the current value displayed in the grid applies to all components associated with the setting.
  • To use the extended settings you must have the component number on hand, there is not a lookup for this value at this time. You can get the component number from one of the following administration tables:
    • In the Form Administration table, the component number for the form is the Template ID #.
    • In the PDQ Administration table, the component number for the PDQ is the PDQ XML ID.
    • In the Status Edit Checks Administration table, the component number for the edit check is the EditCheck ID.
  • The first row of fields are as follow:
    • Sequence Number - If there are multiple extended settings they are processed in Sequence Number order. Initially this field is blank, but when you enter a value into any other field in the section (any field within the yellow background) it automatically populates. If you move the sections up or down, the value in this field automatically updates.
    • ACL (Roles) - Use this field to specify settings based on user role. Leaving it blank means it applies to all user roles.
    • Component Type - Use when you're associating this extended setting with a specific component. Select the type that matches the component. For example, if it's a PDQ, select PDQ from the lookup.
    • Component Number - Use this field to specify settings based on component number. Leaving it blank means it applies to all components.
  • Use the Effective Dates fields to apply start and end dates to extended settings. Both date fields are optional and are not required to be used together. You can have a start date without an end date and vice versa.
  • The Custom Criteria field is for Libera use only.
  • When an extended setting is in use, the Value field within the Extended Settings section is applied to the component or ACL within the section and overrides the global Value setting at the top of the form for that specific component or ACL. The global value still applies to all other components or ACLs.

Test Settings

The Test Settings section can be used to see how the settings in the form are applied by component and/or User.

  • Selecting a Component Type/Component Number is optional. If left blank, the test is run against the user roles of the selected User instead.
  • If you don't select a User other than yourself, it tests the setting against the user roles associated with your logon compared to the user roles, if any, associated with the setting.
  • When testing, the Reference Date is compared against the Effective Dates in the Extended Settings section.
  • Use the Get Value button to show the Value as its applied to the component and/or user selected.
