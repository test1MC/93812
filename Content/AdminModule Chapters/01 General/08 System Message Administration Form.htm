System Message Administration Form


System Message Administration Form.

 

Overview

The System Startup Message Administration Form allows Administrators to create and edit messages available to all users. This serves as a form of "bulletin board" for posting general information to a broad audience. These messages only appear when users log into the system. Administrators can add, modify, and remove messages as desired.

 

 

Adding a New Message

  1. Select the System Administration option on the Application Home Page to load the System Administration Desktop.
  2. Filter the Data Tree by the word start. Select System Startup Messages.
  3. In the ECF, select the Action Row where it says "Please click here to add a new record..." This opens a new System Startup Message Administration Form.
  4. Fill out the form.

 

Editing Existing Messages

  1. In the System Startup Messages ECF, select the message you want to edit. This opens the System Startup Message Administration form.
  2. Edit as desired.

 

Form Functionality

  • Data from all form fields populate the System Startup Messages Administration grid.
  • Users see the Subject as the message header in the System Message window. It appears bolded and underlined above the message.
  • The values for Date Written and Written By appear at the bottom of the message in the System Message window.
  • The Read ACL controls who sees the message. This way you can tailor messages for specific audiences. When left blank, the message appears to all users. The field accepts a comma separated list of User Roles. Optionally, use the Build Read ACL button to create the list.
  • Use the Start and End Date fields to control when the message appears.
  • Persistent means the message doesn't go away, even if the user selects the Have read the information button in the System Message window. It will appear until the set End Date.
  • If the message is not persistent, it appears until the set End Date or if the user selected the Have read the information button.
  • You can deactivate a message at any time by setting Deactivated? to Y (Yes). Deactivated messages do not appear in the System Message window regardless of the message End Date.
  • Use the Type field to categorize the message in the ECF. Message type doesn't change the functionality of the System Message window at all.
  • The Message field is where you type the actual message that users see. You can type as much information as necessary. The text can be formatted.
