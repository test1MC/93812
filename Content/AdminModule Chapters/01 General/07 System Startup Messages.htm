System Startup Messages.

 

Edit Mode: Form and Grid

Description: The System Startup Messages option allows Administrators to create and edit messages available to all application users. This serves as a form of “bulletin board” for posting general information to a broad audience. These message appears upon logging in to the system. The grid shows basic details of these messages, including the subject, text message, start date and end date when the message is displayed, the message type, and whether the message is active (persistent), or is deactivated. This option is designed specifically for agency use. Administrators can add, modify, and remove messages as desired.

 

Adding a New System Startup Message
  1. Select the Action Row where it says, "Please click here to add a new record..." This opens the System Message Administration Form in a new window.
  2. Fill out the form as you would any other form. Refer to System Message Administration Form for more information on working with the System Message Administration Form.


Example of how system messages appear to users.

 

 Editing an Existing System Startup Message
  1. Select an existing system startup message record in the Data Tree. This opens the System Message Administration Form for the selected record in a new window.
  2. Edit the form as you would any other form.

 

 Deleting an Existing System Startup Message
  1. Select the record to be edited, then press F4 or right-click to open the context menu.
  2. Select the Delete option from the context menu. A confirmation message appears, "Are you sure you want to delete this record?" If so, select the OK button in the message window, otherwise select the Cancel button.
  3. Selecting the OK button closes the confirmation message window and deletes the record from the grid.

Note: Instead of deleting an existing system startup message, you can deactivate it by setting the Deactivated field to Y.
