Applying User Roles to Forms

You have the ability to control which users can add a new form, edit a form, or even see the form in the Electronic Case Folder (ECF). You do this via the Form Administration grid in the System Administration module.

Though many of the details of Form Administration are best left to the application developers, other fields are of importance to the agency application administrators. In particular, the Create ACL, Read ACL, and Write ACL fields may need to be updated whenever a new role is added to the system.

  • Create ACL - This setting controls which users have the ability to add a new form to the ECF. If left blank all users can add this form from the list of new forms provided all other form prerequisites have been met. If one or more User Roles exist, then only users with those roles can see the form in the list of new forms provided all other form prerequisites have been met. If a role allows a user to add a new form to the ECF, then they may edit the form. Additional roles may apply to editing the form.
  • Read ACL - This setting controls which users have access to the form in the ECF. If left blank all users can see the form in the ECF and open it for viewing. If one or more User Roles exist, then only users with those roles will see the form in the ECF.
  • Write ACL - This setting controls which users can edit (enter or edit data) on an existing form in the ECF. If left blank all users can open the form and edit it. If one or more User Roles exist, then only users with those roles can edit the form.

Example of how to correctly enter User Roles:

Create ACL: 23,24,1000-1010

This allows users with roles 23, 24, and 1000 through 1010 to add this form to the ECF.

Read ACL: !5

This allows all users except users with role 5 to see the form in the ECF.

Write ACL: 23,24,1000-1010,9888

This allows users with roles 23, 24,1000 through 1010, and 9888 to open this form from the ECF and edit it.

Note: You will find report titles included in the Form Administration grid. Typically, you do not need to edit ACL data for reports here. ACLs get applied to reports in the Report Administration grid.

Important: Libera must be notified of all System Administration changes in a timely fashion to ensure a future update does not overwrite or erase the changes you have made.

  • User Roles do not have to be listed in numeric order, but it does make it easier to view and compare.

  • If an ACL field is blank it means that all users have the ability to perform that particular function i.e. create (add a new blank form to the ECF), read, or write (enter or edit data on a form). Once an ACL has one or more User Roles listed then only users with those roles have the ability to create, read or write.

  • Once you start an ACL you must include all possible roles that should have access to that functionality that you are working on. It's a good idea to always include the System Administrator role:

  • Do not leave two commas together, delete any extra commas.
  • Do not leave a comma at the end of the text.
