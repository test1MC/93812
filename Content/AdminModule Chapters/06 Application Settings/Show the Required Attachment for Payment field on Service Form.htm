Show the 'Required Attachment for Payment' field on the Service form

(App.Service.Field.RequiredAttachmentForPayment.Visible)

 

Controls the Required Attachments for Payment field appearing or not appearing on the form. This is a lookup field where you can select one or more document categories.

Note: This Application Setting is used in conjunction with the Show the Category lookup on attachments Application Setting. Both must be set to true in order to work correctly.

When the Value is set to true, at least one document must be attached to the Authorization form with the specified document category or categories filled in prior to opening the Payment Approval form. If the user attempts to process a payment approval (for a full or partial payment) without the required attachment(s) on file, the following message appears.

An attachment with the category of (X) is required before making a payment on this authorization.

 

This message repeats for each required attachment category until all are attached to the Authorization form before allowing the user to open the Payment Approval form.
