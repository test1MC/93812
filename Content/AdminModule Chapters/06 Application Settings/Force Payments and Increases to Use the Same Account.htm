Force payments and increases to use the same account as the authorization

(App.AccountSelection.UseARecordAccount)

 

When set to true:

  • s will use the following to select accounts:
    • The account must be the same account as the authorization line item associated with that payment
    • The account must not be turned off
    • The Date Expired (Payments) field must be in the future
    • The Available Payment Balance must be high enough to accept the payment
  • If the authorization account is not valid for payment, then it will not let the user pay the authorization.
  • When increasing the amount of an approved and printed authorization, it will try to use the account on the authorization. If this is not valid, it will not allow the user to increase the authorized amount.
  • Additionally, when changing the account on an authorization (either by changing the date and having it automatically calculate the account or by manually changing the account field), it will now move all of the increases to that account also.
  • However, it will not move any existing s to the new account so the accounts for these will need to be edited manually if needed. Any payments made after the account change will use the new account.

When set to false:

  • Account selection functionality remains unchanged.
