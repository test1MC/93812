Enforce Date Expired for account selection on Payment Approvals

(App.PaymentApproval.Account.EnforceDateExpired)

 

If true, the Date Expired (Payments) field on an account must be in the future for it to be valid for the form. If false, the field is ignored when determining valid accounts.

User Role Able to make payments using expired accounts (30012100) can be used to allow specific users to use expired accounts if this application setting is set to false.

Note: The service rendered/provided date still has to be within the date range of the account. The exception to this is if the application setting Force payments and increases to use the same account as the authorization is set to true, in which case it will use the account on the authorization regardless of the date entered on the form.
