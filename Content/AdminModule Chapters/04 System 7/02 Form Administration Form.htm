Form Administration Form


Portion of the Form Administration form.

 

Overview

The Form Administration form is used to edit information about the selected form such as Form Name, Description, and ACLs. Only users with access to the System Administration module have access to this form.

 

 

Fields Maintained by the Agency

If you change any information in Form Administration, please inform your Libera Project Coordinator, so the environments can be kept in sync.

  • Form Name - This is the name of the form as displayed in the File Drawer and Electronic Case Folder (ECF).
  • Description - This is the description that appears in the File Drawer.
  • Security Access - ACLs - Access Control List (ACL). They are used to define who has rights to the component and exactly which rights - Read, Create, Write, and/or Delete. You may use the Build ACL buttons or type comma separated lists in the fields.
    • Read ACL - Users that have any of the user roles in this field have the ability to see this form in the ECF. They can open it and see the contents of the form.
    • Create ACL - Users that have any of the user roles in this field have the ability to see this form in the File Drawer and add it as a new form to the ECF.
    • Write ACL - Users that have any of the user roles in this field have the ability to edit the form.
    • Delete ACL - Users that have any of the user roles in this field have the ability to delete the form from the ECF. Some forms are not configured to be deleted. Please contact Libera for assistance with removing the form.

 

Fields Maintained by Libera

The following fields listed below are maintained by Libera and should not be changed by the Agency. Explanations of the fields are provided for your information only. Please direct questions about these fields to your Libera Project Coordinator.

  • Form Number - Also known as Template ID and Component Number. This is the unique ID for the form component.
  • XML File - Think of this as the "address" of the component.
  • Single Instance - Allows the form to be added to the ECF multiple times or, not.
  • Days Between Instances - Related to Single Instance, if multiple instances of a form are allowed this setting controls the frequency.
  • Agency Header - Determines if the Agency's header appears on the form.
  • Agency ID - Not used at this time.
  • Module Name - This controls when the component is visible in the File Drawer. For example when the value is Case Management, the component appears when a Case Management PDQ is loaded in the Data Tree.
  • Program - This associates the form with a program.
  • Min/Max Status - Controls when the form can be added to the ECF.
  • Note Type - Depending on the form, this is the code that shows in the Type column of the ECF (if the Type column is visible in the ECF for your Agency).
  • Stage - Used by Libera to control when forms become available to the Agency. Think new Agencies or new programs to an Agency.
  • Form Arguments - Code that controls all kinds of things including but not limited to: how/when the form displays, layout/look of form, and more.
  • Excluded Status(es) - Prevents the form from being visible in the File Drawer when the case is in an excluded status.
  • Form Precedence - Used to set up forms required to be on file prior to allowing this form to be added to the ECF.
  • Universal Styling Settings - Used to control the look and feel of forms specifically related to HTML/XML. This is controlled globally. However, these fields can be used to make this form look different from other forms.
  • Extended Settings - These are the application settings that affect this form.
  • Forms to Launch - This is where we could link other forms to this form. They can be set to automatically open, appear in the ECF, or display a list for the user to select from.
  • System Variable Date Mode - Tells the system which date to look at when applying application settings.
  • Form Properties - A listing of application settings associated with the form.
  • External Form Content - Used for PDF forms.
