Report Administration



Report Administration.

 

Edit Mode: Form

Description: The Report Administration option allows Administrators to manage reports available in the application. The reports are listed in a grid, grouped by Module and Report Category. Do not add or remove any reports, and the form or PDQ reference for the report should remain unchanged. However, the agency can modify the name, description, module, and category of the reports, and the User Roles (Create ACL).

 

ECF Columns

  • ID - Also known as Report #. This is the unique identifier for the report.
  • Name - Name of the report. It also appears in the Reports tab of the File Drawer.
  • Description - A brief description of the report. It also appears in the Reports tab of the File Drawer.
  • Filename - Another ID number. Usually matches the value in the ID column. In some older reports it might different.
  • Type - Tells you the type of report output. Most commonly used types are form (1) and PDQ (2).
  • Module Name - The module with which the report is associated. Controls when the report is available in the File Drawer. For example, if the report is associated with Vendors and you are working in the Case Management module, you won't have access to the report.
  • Report Category - Categorizes the report within the module. The report will be found grouped under this category.
  • Create ACL - Comma separated list of User Roles that have the ability to see and run the report.
  • Flags - Essentially, this is a "tag" field for Libera developers. Any value in this field is informational only and doesn't impact report functionality in any way. A common usage example is that it displays a parent PDQ or file, if any. This functionality is managed by Libera. Do no modify.
  • Arguments - Controls all kinds of things including but not limited to: how/when the report displays, layout/look of the report, and more. This functionality is managed by Libera. Do not modify.
  • Parameters - Displays report parameters, if any. Parameters may be visible to the user or handled in the background. This functionality is managed by Libera. Do not modify.
  • Stage - Controls when the report is visible to the Agency. Note: User Roles also control who can see and use the report, so even if the report is visible, it may not be available to all users.

Adding or Deleting Reports

Libera manages adding and deleting of reports. While you may have the ability to technically add or delete reports, you should not. You would like a report added or deleted please contact your Supervisor or Libera Client Support.
