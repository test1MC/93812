Dashboard Administration Form


Top of the Dashboard Administration form.

 

Overview

The Dashboard Administration form allows you to configure dashboard Items. You can access this form from the System Administration module or via the Gear button (visible to System Administrators) associated with each Item on the dashboard.

 

 

Form Fields
  • Name - Type the name of the Item here. This name will appear on the Item when it is added to the dashboard.
  • Category - To add the Item to a category, press the Search Existing Categories Button to the right of this field. This will open the list of existing categories. one. The Category field will be filled with the one you choose and the Item will be added. If the category you wish to add the item to does not already exist, you must create a new one by typing the desired name of the category by hand into the Category field. The new category will be added as soon as the cursor focus leaves the field and it will be available in the lookup for future Dashboard items. Note: If all of a category's Items are moved out of it, it will no longer be available in the lookup and must be manually added again.
  • Type - The type determines how the information provided by its PDQ is displayed on the Dashboard. Available types:
    • Bullet - Has a box and a horizontal bar.
    • Chart - Displays chart.
    • Gauge - Has an arch graphic reminiscent of a dashboard.
    • Table - Displays a table.
      Refer to the Dashboard topic for more information on the available types.
  • Scope Type - Controls performance for when the value is calculated from system data. If the Item is a "statewide" statistic, meaning that only users with a statewide role are allowed to see this Item, the Scope Type should be set to global. When it's set to global, the value is calculated once regardless of the number of users subscribed to the Item because they all see the same information. If the Item is based on the individual user, the Scope Type should be set to user. The assumption is that the value will be different for every user and the value must be calculated individually for each subscribed user.
  • Statistic Data Type - The only configured option as of 08/21/2013 is scalar. This option returns one number, the current value, that is displayed in the graphic of the widget. For example, in a gauge Item, this is the number displayed centered below the arch.
  • Cache Type - This setting controls where the system stores information to display in the Dashboard. As of 08/21/2013, please only use the dashboard option as the others are not configured for use yet.
  • Default, Minimum, and Actual Refresh Intervals - The Default Refresh Interval sets the refresh interval of the item when a user first adds it to their dashboard. The user can adjust this interval, but no user can set it lower than the value in the Min Refresh Interval field. The Actual Refresh Interval is the lowest interval currently being used by any user. There are two kinds of refreshing happening:

Screen Refreshing

This is what causes the dashboard to blink/flutter and display new data, if available. It is separate from the refresh fields described above. It's controlled by the Interval (mins) column in the Select Dashboard Items PDQ.

All users can edit the value in this column via the grid context menu. While you can set a different value for every option, the screen refreshes at the minimum interval of all selected Items. This affects only the individual user. All users can have different screen refresh rates.

Data Refreshing

This is related to the scheduled task that controls when new data is displayed in the Dashboard for all users. This is the Actual Refresh Interval.

For example:

You set the Interval (mins) for one of your Items to refresh every one minute. You have two widgets on your dashboard. Widget-01 is set to refresh hourly and Widget-02 is set to refresh every four hours.

This means that your dashboard refreshes every minute, but Widget-01 will show new data hourly and Widget-02 will show new data every four hours. In this example, it might be more productive to set the screen refresh interval to one hour.

  • Last Refresh - This is the last date and time the information for this item was updated for any single user. Some users may still have only older data visible on their dashboards until the next screen refresh.
  • Statistics PDQ - This is the PDQ which generates the data for the Item. This PDQ is never visible to end users.
  • Statistics PDQ Params - Works with the PDQ Display Params and is often the same. Provides parameters for the statistic. No parameter window is displayed to the user. Libera maintains this field.
  • PDQ Display - Type the component number of the PDQ that will be opened in a new window by clicking the item on the dashboard. You can use this value to filter the PDQ Administration grid to see the name of the PDQ and more detailed administrative information about the PDQ. Libera maintains this field.
  • PDQ Display Params - Works with the Statistics PDQ Params and is often the same. Provides parameters for the PDQ display. No parameter window is displayed to the user. When the Item name is selected in the Dashboard, the PDQ opens in a new window displaying data based on the parameters set in this field. This field also uses information provided from the Program field. For example, if the Item is providing eligibility statistics, the Program field determines which program these statistics come from. If the Program field is changed, the Item's Name and Category should also be changed to match. Libera maintains this field.
  • Program - Controls the program associated with the statistic displayed in the Item. This should be correlated with the Item's Name and Category.
  • Number of Retention Days - Specifies how far back data points are stored. If empty, the default value is one year. This affects data displayed in charts.
  • Goal Value - This field allows users to do additional processing on the returned value to determine statistics like percentages or over/under values.
  • Libera Developers maintain the following fields:
    • Default Value
    • Dataformat
    • Calculation Type
    • Expression
    • Filter Expression
    • Data Command XML

 

 Security Access


Security Access section.

  • View ACL - Enter a comma-separated list of the roles that are allowed to add this item to their Dashboards. If the field is blank, all users will have access to it.
  • Forced to view item ACL - Enter a comma separated list of the roles that will have this Item appear on their dashboard by default. Once this is set, users may be able to temporarily remove this from their dashboard. However, when they close and reopen their Dashboard, the Item will automatically appear.
  • Admin ACL - Enter a comma-separated list of the roles that are able to edit the properties of this item. If the field is blank, all users will have access to it.

 

 Threshold Settings


Threshold Settings section.

Setting thresholds are a way to let users know which items are high priority. They assign a Severity to the item when the data provided by the item is within a certain range.

An item can have several thresholds with different levels of Severity. Please refer to the Multiple-Section Sections topic for additional information on adding multiple sections.

For example, an item that tracks the number of clients overdue for follow ups may have one threshold with a Severity of "Normal" for when there are five or fewer such clients, another with a Severity of "Warning" for when there are between five and ten, and a third threshold with a Severity of "Severe" for when there are more than ten such clients.

  • Numeric - Numeric means that the threshold levels are saved as numeric values. If this field is unchecked, the user has the option to save the thresholds as alphanumeric characters.
  • Min Threshold - This is the lower limit of the threshold. To avoid overlapping, it should be greater than the Max Threshold of the previous threshold, if any.
  • Max Threshold - This is the upper limit of the threshold.
  • Severity - Select a rating for the severity of the item when its value is within this threshold.
  • Threshold Type - Select the threshold type here. The type of the threshold determines if a notification should be sent to the user once the threshold is reached and the user is subscribed to the Item.

 

 Subscribed Users


Subscribed Users section.

A grid appears when expanded which displays the users who have subscribed to this Item. It also displays their Interval value for the item. You can add new users via the Action Row. You can also delete a user or edit the Interval via the context menu in the grid.

Use the Calculate New Value for Dashboard button to force the Item to refresh the data displayed for all users currently subscribed to it.
