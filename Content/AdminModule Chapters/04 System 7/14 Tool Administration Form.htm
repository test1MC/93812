Tool Administration Form


Tool Administration form.

 

Overview

The Tool Administration form is used to edit information about the selected tool such as name and ACL.

 

 

Form Fields

Fields Maintained by the Agency

  • Use the ACL (Roles) field to control which groups of users have access to the tool via the Tools tab of the File Drawer.
  • The process is the same as it is for forms and PDQs. You may use the Build ACL button or type a comma separated list.

 

Fields Maintained by Libera

The following fields listed below are maintained by Libera and should not be changed by the Agency. Explanations of the fields are provided for your information only. Please direct questions about these fields to your Supervisor or Libera Client Support.

  • Tool # - This is the unique ID number for the tool component.
  • Name - This is the name of the tool as displayed in the Name column of the Tool Administration table and the Tools tab of the File Drawer.
  • Group - This is used to group like tools together in the Tool Administration table and the on the Tools tab of the File Drawer.
  • Module Name - This controls when the tool is visible on the Tools tab. For example, when the value for this field is General, the tool is available all of the time, however if it is anything else, the tool will only appear when the selected module is loaded in the Data Tree.
  • Agency ID - No longer used.
  • Object Type - Typically related to the Module, used more with form components than with tools.
  • Flags - Component specific instructions. Used more with other types of components such as forms and PDQs than with tools.
  • The Javascript Code and XML Data Commands fields provide specific instructions for the component relating to many things including how/when it opens, window size, etc.
