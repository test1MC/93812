Status Edit Checks Administration.

 

Edit Mode: No editing

Description: The Status Edit Checks Administration option allows Libera Developers to manage the edit checks executed when a status change occurs. Each edit check is listed in the grid.

Libera recommends that only Libera Developers maintain this list, as errors in the configuration can lead to application errors and permit invalid data. Important: Entering invalid data can result in reporting errors. If you have any questions about how status edit checks work with your application, please contact your Supervisor or Libera Client Support.

 

ECF Columns

  • Program - Program with which the status edit check is associated. Note: one status edit check can be associated with many programs.
  • Status From - Current status. If you filter by Program and then by Status From you'll see all possible edit checks that can run when moving out of the selected status and program.
  • Status To - Destination status from the Current Status in the same row.
  • EditCheck Description - Name of the edit check. It often refers to the field which is impacted by the edit check or describes the main topic of the edit check.
  • EditCheck ID - Unique identification number associated with the edit check.
  • Order - It's possible for one status change to trigger multiple edit checks. The value in this column indicates the order in which the edit checks run.
  • Group # - Controls the order in which the edit check is run. There are two options. All ECIDs with 2001 are run before those with 9901.
  • Enabled - Check box indicating if the edit check is enabled for your Agency or not. Note: Disabling any edit check could result in disastrous consequences for your Agency's data. If you would like an edit check enabled or disabled, please contact your Supervisor or Libera Client Support.
  • Confirm - Displays confirmation message for the edit check.
  • Entity Type - Indicates the entity to which the edit check is associated. For example, client, unit, building, etc.
