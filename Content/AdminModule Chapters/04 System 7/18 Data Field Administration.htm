Data Field Administration



Data Field Administration.

 

Edit Mode: No editing

Description: The Data Field Administration option allows Libera Developers to create special access rules for specific data. Reading and writing (including the initial write) can be limited to specific user roles and/or edit checks. Some functionality in the System 7™ application needs to load and save data from a specific column in a specific database record. Using the data in this grid, System 7™ can read and write the data to the specific database record. The grid shows a list of all data fields with special access rules, including information identifying the field within the application database and a list of roles with access.

Important: This functionality requires knowledge of the underlying database and the profile used to access it. While the grid columns are editable, Libera recommends that Administrators do not make changes to this grid.

 

ECF Grid Columns

  • ID – ID number for the new data field.
  • DbProfile – Database profile name.
  • DbTable – Name of the database table.
  • Key Column – Name of the primary key column for this table.
  • Data Column – Name of the data column for this table.
  • Data Format – Format of the data column.
  • Description – Description of the data field.
  • Read ACL – ID numbers for roles that can view the data field.
  • Write ACL – ID numbers for roles that can write information to the data field.
  • WriteFirst ACL – ID numbers for roles that can write information to the data field for the first time (if original value was empty).
  • Remove ACL – ID numbers for roles that can remove a data field value (currently only applies to Document Management attachments).
  • Flags - Developer notes.
  • Read EditCheck – ID numbers for roles that can view Edit Check.
  • Write EditCheck – EditCheck ID to call when the data field value is about to be written (Development Use Only).
  • Document Storage Tier – The server tier where the attachment is stored. Attachments are currently stored in the Web Server or the Application Server. For database storage, the Application Server tier is used.
  • Document WebServer Storage Path – The path where documents are stored when Web Server is specified as the storage tier.
  • Document AppServer Storage Path – The path where documents are stored when Application Server is specified as the storage tier.
  • Allowed Document Mime Type – Zero (0) or more Mime Types that may be stored in this data field.
  • GetDoc Web EC – The EditCheck ID to call when getting a document in the Web tier.
  • GetDoc App EC – The EditCheck ID to call when getting a document in the Application tier.
  • SaveDoc Web EC – The EditCheck ID to call when saving a document in the Web tier.
  • SaveDoc App EC – The EditCheck ID to call when saving a document in the Application tier.
  • RemoveDoc Web EC – The EditCheck ID to call when removing a document in the Web tier.
  • RemoveDoc App EC – The EditCheck ID to call when removing a document in the Application tier.
